## CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Hugo static site for FantasyMaps (deployed to `www.fantasymaps.org` via the `static/CNAME` file). Source lives on `main`; the `gh-pages.yml` workflow builds with the Hugo extended toolchain and publishes `./public` to GitHub Pages on every push to `main`.

## Commands

- `hugo server -D` — local dev server with drafts enabled.
- `hugo --minify` — production build into `./public` (matches CI).
- `hugo new <section>/<name>.md` — scaffold content using `archetypes/default.md` (creates a draft).

The CI workflow expects Hugo **extended** (`peaceiris/actions-hugo@v2` with `extended: true`) — install the same locally to avoid SCSS/asset divergence.

## Theme submodule

`themes/ananke` is a git submodule (`https://github.com/theNewDynamic/gohugo-theme-ananke.git`) and is **not** checked in — the directory is empty until initialized. After cloning, run `git submodule update --init --recursive` or Hugo will fail to find the theme declared in `config.toml`.

## Layout overrides

`config.toml` sets `theme = "ananke"`, but local files in `layouts/` override theme files of the same path:

- `layouts/_default/baseof.html` — the site's root template; gates indexing and Google Analytics on `HUGO_ENV=production` (or `Site.Params.env=production`). Set `HUGO_ENV=production` when building anything intended for indexing.
- `layouts/partials/site-footer.html`, `layouts/partials/social-follow.html` — override ananke's equivalents.
- `layouts/shortcodes/rawhtml.html` — used inside content (e.g., the Patreon button on the homepage) to embed unescaped HTML; prefer it over disabling Hugo's safe-HTML defaults.

When editing presentation, check whether the corresponding file already exists under `layouts/` (override) before touching the submodule.

## Content structure

- `content/_index.md` — homepage copy.
- `content/tools/`, `content/conference/` — section pages; `sectionPagesMenu = "main"` in `config.toml` auto-adds sections to the nav.
- `content/contact.md` — standalone page.
- The `[[menu.main]]` block in `config.toml` adds external links (e.g., `map.fantasymaps.org`) alongside the auto-generated section entries.

## Build output

`public/` is committed in this repo but is regenerated by CI on every push — treat it as a build artifact, not a source of truth. Don't hand-edit files there.

## Design Context

Canonical source: `.impeccable.md` at the repo root. Summary below for any design or frontend work in this repo.

### Users (in priority order)
1. **Players** — story-first folks deciding whether they want to inhabit this fantasy world. Often on a phone, often at night. The site is a *cover and first plate of an atlas*, not a product page.
2. **Game masters** — Foundry VTT users scanning for proof the tools work.
3. **AI tinkerers** — drawn in by the MCP server / OHM stack; need legitimacy signals, not pitch.
4. **Patrons / investors** — the project should look serious and ongoing without ever pivoting into a deck.

### Brand
**Crafted, warm, expeditionary.** Voice is an open-source explorers' guild — a small research lab that loves dragons. Confident, not solemn. The "Serious tech for non-serious worlds" tagline is the *only* place the design is allowed to wink.

NOT: a SaaS platform, a corporate page, a metal album cover, a children's storybook, a Hugo-default theme.

### Aesthetic
**Paper page, instrument-panel maps.** Body lives on warm off-white paper; map / tool / demo surfaces are framed in deep teal-navy "chart room" panels so the maps glow. The contrast between the two surfaces is the signature.

References (mood, not pastiche): old USGS topo sheets, naval charts, Tufte books, field-research journals, atlas plates with marginalia.

Anti-references: SaaS gradient/card/hero look, blackletter heavy-metal fantasy, Hugo / Ananke default silhouette.

### Principles
1. **Players first; lore is the headline.** Lead with the world, not the tech. Tech credibility goes underneath as proof.
2. **The page is a printed plate.** Confident asymmetry, marginalia, captions, plate numbers. Reject centered-hero / 3-card-grid silhouettes.
3. **Maps glow; prose breathes.** Two-surface system. Color and motion concentrate inside chart-room panels.
4. **Every ornament has a job.** Decoration must double as information (a coordinate, a compass mark, a plate number). No purely decorative gradients, glassmorphism, or icon-above-heading templates. Never `border-left: Npx solid` accent stripes.
5. **Don't be in on the joke too loudly.** Restraint is what separates this project from every generic fantasy site.

### Type & Color (quick rules — full direction in `.impeccable.md`)
- **Headings**: humanist serif with character — Spectral / Eczar / Vollkorn / Bitter. NOT Crimson, Fraunces, Newsreader, Playfair, Cormorant.
- **Body**: warm low-contrast humanist sans — Alegreya Sans / Public Sans / Figtree. NOT Inter, DM Sans, Plus Jakarta, Outfit, Manrope.
- **Mono**: Departure Mono or Commit Mono, only where it carries information. NOT IBM Plex / Space / JetBrains Mono.
- **Color**: OKLCH throughout. Paper `oklch(97% 0.012 85)`, ink `oklch(22% 0.02 50)`, cinnabar accent `oklch(58% 0.17 32)` (used sparingly), chart-room surface `oklch(22% 0.04 230)`, hairlines `oklch(85% 0.008 80)`. Never pure black or pure white.
